# NURI Base 검색 (Vault Search)

NURI Base 전체를 대상으로 다층 검색을 수행합니다.

## 트리거 조건

- "찾아줘", "관련 노트", "뭘 정리했었지?"
- "~에 대해 내가 쓴 거 있어?"
- "이 주제 관련 노트 모아줘"

## 검색 계층

### 1단계: 키워드 검색 (빠름)

```bash
# 파일명 검색
find notes/ -name "*.md" -iname "*검색어*"

# 내용 검색
grep -rl "검색어" notes/ --include="*.md"

# 태그 검색
grep -rl "tags:.*검색어" notes/ --include="*.md"
```

### 2단계: 프론트매터 필터링

```bash
# 특정 유형의 노트만
grep -rl "type: permanent" notes/ --include="*.md"

# 특정 날짜 범위
find notes/ -name "*.md" -newermt "2026-01-01" -not -newermt "2026-04-01"

# 특정 태그 조합
grep -rl "tags:.*태그1" notes/ --include="*.md" | xargs grep -l "tags:.*태그2"
```

### 3단계: 맥락 검색 (시맨틱)

키워드로 찾기 어려운 경우, 관련 노트의 내용을 읽고 의미적 연관성을 판단합니다:

1. 1단계 검색으로 후보 노트 목록을 확보합니다 (최대 20개)
2. 각 후보의 내용을 `read_file`로 읽습니다
3. 검색 의도와의 관련성을 분석합니다
4. 관련성 순으로 정렬하여 결과를 제시합니다

### 4단계: 세션 검색 결합

NURI Base 파일 검색과 Hermes의 FTS5 세션 검색을 결합합니다:

```
session_search 도구로 과거 대화에서 관련 내용 검색
+ NURI Base 파일 검색 결과
= 통합 맥락 요약
```

## 결과 형식

검색 결과는 다음 형식으로 제시합니다:

```markdown
## 검색 결과: "검색어"

총 N개 노트에서 관련 내용을 찾았습니다.

### 가장 관련 높은 노트

1. **[[노트-제목-1]]** (permanent, 2026-03-15)
   > 관련 내용 발췌 (1~2문장)
   태그: #태그1 #태그2

2. **[[노트-제목-2]]** (literature, 2026-02-20)
   > 관련 내용 발췌
   태그: #태그3

### 관련 대화 이력

- 2026-03-10 세션: "이 주제에 대해 ~라고 논의함"
- 2026-02-15 세션: "~를 정리하면서 ~를 발견"

### 연관 주제 (탐구 제안)

- "관련주제A" — [[노트-1]]에서 언급되었으나 별도 노트 없음
- "관련주제B" — [[노트-2]]와 연결 가능
```

## 검색 팁

| 검색 요청 | 수행 방식 |
|----------|----------|
| "최근 작성한 AI 관련 노트" | 날짜 필터 + 키워드 |
| "분산 시스템이랑 합의 알고리즘 연관된 것들" | 다중 키워드 + 맥락 분석 |
| "지난달에 논의했던 아키텍처 결정" | 세션 검색 + 파일 검색 결합 |
| "영구 노트 중에서 머신러닝 관련" | 프론트매터 type 필터 + 키워드 |

## 주의사항

- 검색 결과가 많을 때는 상위 10개만 보여주고 "더 보기" 옵션을 제시합니다
- 검색어를 동의어/관련어로 확장하여 누락을 줄입니다
- 결과에 MOC 파일이 포함되면 우선적으로 안내합니다 (진입점 역할)
